## 亮点一：能力交集算法 — 最小权限原则的数学实现

### 问题

当 Agent A 委派给 Agent B 时，B 的有效权限如何确定？传统做法是直接继承父权限，但这违反最小权限原则——子Agent可能获得超出其声明的权限。

### 解决方案

三级能力计算策略：**交集优先 → 通配符降级 → 委派权限兜底 → 拒绝**

### 核心代码

```python
# app/delegation/engine.py — delegate() 方法

def delegate(self, parent_token, target_agent, action, caller_agent=""):
    parent_caps = set(parent_claims.get("capabilities", []))
    target_caps = set(CAPABILITY_AGENTS[target_agent]["capabilities"])

    # 分离委派权限和操作权限
    parent_action_caps = {c for c in parent_caps if not c.startswith("delegate:")}
    target_action_caps = {c for c in target_caps if not c.startswith("delegate:")}
    target_delegate_caps = {c for c in target_caps if c.startswith("delegate:")}

    # 三级能力计算
    action_overlap = parent_action_caps & target_action_caps
    if action_overlap:
        effective_caps = action_overlap          # ① 有交集 → 取交集（最安全）
    else:
        parent_wildcard = any(c == "*" or c.endswith(":*") for c in parent_action_caps)
        if parent_wildcard:
            effective_caps = target_action_caps  # ② 父有通配符 → 取子集
        elif has_delegate_cap:
            effective_caps = target_action_caps  # ③ 有委派权限 → 取子集
        else:
            return DENIED                        # ④ 无交集且无委派权限 → 拒绝

    effective_caps = effective_caps | target_delegate_caps
```

## 亮点二：三层融合输出对齐检测 — 首个Agent输出安全引擎

### 问题

现有AI安全只关注输入（Prompt Injection），完全忽略输出安全。Agent输出可能：
1. 偏离原始任务目标（目标劫持）
2. 包含外部注入的恶意指令（间接注入）
3. 泄露敏感数据（DLP违规）

### 解决方案

三层加权融合检测 + 交叉放大机制：

```
目标劫持(40%) + 间接注入(35%) + DLP泄露(25%) → 综合风险评分
                                              ↓
                              劫持+注入同时触发 → ×1.3 交叉放大
                                              ↓
                    risk ≥ 0.7 → BLOCK | risk ≥ 0.4 → WARN | risk < 0.4 → ALLOW
```

### 核心代码

```python
# app/security/alignment.py — AlignmentCheckEngine

class AlignmentCheckEngine:
    def check(self, original_message, agent_output, context=None):
        # 第1层：目标劫持检测（权重40%）
        hijack = self._check_goal_hijack(original_message, agent_output)

        # 第2层：间接注入检测（权重35%）
        injection = self._check_indirect_injection(agent_output)

        # 第3层：DLP敏感信息检测（权重25%）
        dlp = self._check_dlp_output(agent_output)

        # 综合风险评分
        risk = self._compute_risk(hijack, injection, dlp)

        # 三级处置策略
        if risk >= 0.7:
            return AlignmentResult(aligned=False, action="block")
        elif risk >= 0.4:
            return AlignmentResult(aligned=True, action="warn")
        else:
            return AlignmentResult(aligned=True, action="allow")

    def _compute_risk(self, hijack, injection, dlp):
        h_score = hijack.get("score", 0.0) * 0.40
        i_score = injection.get("score", 0.0) * 0.35
        d_score = dlp.get("score", 0.0) * 0.25
        combined = h_score + i_score + d_score

        # 交叉放大：劫持+注入同时触发 → 风险×1.3
        if hijack.get("detected") and injection.get("detected"):
            combined = min(1.0, combined * 1.3)

        return round(min(1.0, combined), 4)

    def _check_goal_hijack(self, original_message, agent_output):
        # 10组正则模式匹配（指令覆盖、角色变更、记忆清除等）
        pattern_hits = [atype for p, atype in self._compiled_hijack if p.search(output_lower)]

        # 语义漂移计算（Token重叠度 → 余弦相似度）
        semantic_drift = self._compute_semantic_drift(msg_lower, output_lower)

        score = 0.4 * len(pattern_hits) + semantic_drift * 0.5
        return {"detected": score >= 0.4, "score": min(1.0, score)}

    def _check_indirect_injection(self, agent_output):
        # 12组正则模式（钓鱼链接、代码执行、凭证窃取等）
        pattern_hits = [atype for p, atype in self._compiled_injection if p.search(output_lower)]

        # URL计数 + 代码块计数增强
        score = 0.4 * len(pattern_hits) + 0.1 * url_count
        return {"detected": score >= 0.4, "score": min(1.0, score)}

    def _check_dlp_output(self, agent_output):
        # 调用DLP引擎 + 6类敏感模式匹配 + 自动脱敏
        dlp_result = DLPEngine().check(agent_output)
        for stype, pattern in self._compiled_sensitive.items():
            matches = pattern.findall(agent_output)
            if matches:
                masked_text = masked_text.replace(m, m[:4] + "****" + m[-4:])
        return {"leaked": score >= 0.4, "masked_text": masked_text}
```

## 亮点三：动态信任评分与自动撤销 — 行为驱动的权限收缩

### 问题

传统IAM的权限是静态的——一旦授予就不会改变。但AI Agent的行为是动态的，一次越权尝试就应该立即收缩权限。

### 解决方案

不对称惩罚机制 + 自动撤销 + 冷却恢复：

```
操作成功 → 信任分 +0.01（缓慢奖励）
策略拒绝 → 信任分 -0.05（中等惩罚）
越权尝试 → 信任分 -0.15（严厉惩罚）

信任分 < 0.3 → 自动撤销所有Token + 信任分归零
5分钟冷却后 → 恢复至阈值0.5（不是原始值，防止反复攻击）
```

### 核心代码

```python
# app/delegation/engine.py — 信任分动态调整与自动撤销

TRUST_REWARD = 0.01           # 操作成功 +0.01
TRUST_PENALTY_DENY = 0.05     # 策略拒绝 -0.05
TRUST_PENALTY_ESCALATION = 0.15  # 越权尝试 -0.15
AUTO_REVOKE_THRESHOLD = 0.3   # 自动撤销阈值
TRUST_COOLDOWN_SECONDS = 300  # 冷却恢复时间

def update_trust_score(agent_id, delta):
    before = AGENT_TRUST_SCORE.get(agent_id, 0.5)
    after = max(0.0, min(1.0, before + delta))
    AGENT_TRUST_SCORE[agent_id] = after
    _persist_trust_score(agent_id, after)

    # 信任分低于阈值 → 自动撤销
    if after <= AUTO_REVOKE_THRESHOLD and agent_id not in AUTO_REVOKED_AGENTS:
        auto_revoke_agent(agent_id, reason=f"Trust score {after:.2f} < threshold")
        return before, after, True  # auto_revoked=True

    return before, after, False

def auto_revoke_agent(agent_id, reason=""):
    # 1. 撤销该Agent所有Token
    revoke_tokens_by_agent(agent_id, reason)
    # 2. 信任分归零
    AGENT_TRUST_SCORE[agent_id] = 0.0
    _persist_trust_score(agent_id, 0.0)
    # 3. 加入自动撤销记录
    AUTO_REVOKED_AGENTS[agent_id] = {
        "reason": reason,
        "trust_score_at_revoke": trust_before,
        "revoked_at": datetime.now(timezone.utc).isoformat(),
    }

def try_cooldown_recover(agent_id):
    # 冷却期未过 → 拒绝恢复
    if elapsed < TRUST_COOLDOWN_SECONDS:
        return {"recovered": False, "cooldown_remaining": remaining}
    # 冷却期过后 → 恢复至阈值（不是原始值）
    AGENT_TRUST_SCORE[agent_id] = TRUST_THRESHOLD  # 0.5，不是0.9
    del AUTO_REVOKED_AGENTS[agent_id]
    return {"recovered": True, "trust_score": TRUST_THRESHOLD}
```

## 亮点四：六层纵深安全验证 — 每个请求的实时安全屏障

### 问题

单层安全检查容易被绕过。需要一个纵深防御体系，任何一层被突破后还有后续层拦截。

### 解决方案

六层验证流水线，每个请求实时执行，平均延迟 < 15ms：

```
L1 身份认证 → L2 能力验证 → L3 委派链验证 → L4 行为检测 → L5 运行时控制 → L6 可观测性
     ↓             ↓              ↓              ↓              ↓              ↓
  Ed25519/JWT   RBAC+ABAC     chain_depth    注入检测+      阻断检查+      审计记录+
  指纹验证      策略评估       无环检查       自动撤销       预算检查       哈希链验证
```

### 核心代码

```python
# app/security/six_layer_verify.py

def verify_six_layers(agent_id, action, input_text, trust_score, risk_score, ...):
    layers = []

    # L1: 身份认证 — Ed25519指纹 / JWT+HMAC
    layers.append(_verify_identity(agent_id))

    # L2: 能力验证 — 声明式策略评估 + RBAC回退
    layers.append(_verify_capability(agent_id, action, role, trust_score))

    # L3: 委派链验证 — 深度检查 + 无环检查
    layers.append(_verify_chain(agent_id, delegation_chain))

    # L4: 行为检测 — 注入模式匹配 + 自动撤销检查 + 风险评分
    layers.append(_verify_behavior(agent_id, input_text, risk_score, trust_score, auto_revoked))

    # L5: 运行时控制 — 阻断检查 + 预算检查 + 降权执行 + 节流
    layers.append(_verify_runtime(agent_id, action, trust_score, risk_score, blocked_at, allowed))

    # L6: 可观测性 — 审计记录 + 哈希链完整性验证
    layers.append(_verify_observability(request_id, agent_id, action))

    # 综合判定
    fail_count = sum(1 for l in layers if l.status == "fail")
    warn_count = sum(1 for l in layers if l.status == "warn")

    if fail_count > 0:   overall_status = "BLOCKED"
    elif warn_count > 0: overall_status = "DEGRADED"
    else:                overall_status = "SECURE"
```

## 亮点五：三Agent异构协作编排 — 容错降级的安全协作

### 问题

多Agent协作时，如何确保每步委派都经过安全验证？子Agent失败时如何不阻塞整体流程？

### 解决方案

独立Token签发 + 容错降级 + 全链路审计：

```
Step 1: doc_agent → data_agent（独立root_token_1）
Step 2: doc_agent → external_agent（独立root_token_2）
Step 3: doc_agent 汇总报告（含降级处理）
```

### 核心代码

```python
# app/orchestrator/orchestrator.py — _handle_collaborative_task()

def _handle_collaborative_task(user_id, message, engine, platform, ...):
    # Step 1: doc_agent → data_agent（数据查询）
    root_token_step1 = engine.issue_root_token(
        agent_id="doc_agent", delegated_user=user_id,
        capabilities=["read:doc", "delegate:data_agent", "delegate:external_agent"]
    )
    step1_result = secure_agent_call(
        engine, root_token_step1, "doc_agent", "data_agent", "read:feishu_table:finance"
    )

    # Step 2: doc_agent → external_agent（外部搜索）— 独立Token
    root_token_step2 = engine.issue_root_token(
        agent_id="doc_agent", delegated_user=user_id,
        capabilities=["read:doc", "delegate:data_agent", "delegate:external_agent"]
    )
    step2_result = secure_agent_call(
        engine, root_token_step2, "doc_agent", "external_agent", "read:web"
    )

    # Step 3: doc_agent 汇总报告（含降级处理）
    has_degradation = data_fallback or external_fallback
    overall_status = "degraded" if has_degradation else "success"

    # 全链路审计
    full_chain = ["user:" + user_id, "doc_agent", "data_agent", "external_agent", "doc_agent"]
    _log_event(user_id, message, "doc_agent", "collaborative:full_chain", overall_status, ...)
```

def report_failure(agent_id, error_type, severity="medium"):
    severity_thresholds = {"low": 5, "medium": 3, "high": 2, "critical": 1}

    if agent_id not in _AGENT_HEALTH:
        _AGENT_HEALTH[agent_id] = {"failure_count": 0, "circuit_open": False}

    health = _AGENT_HEALTH[agent_id]
