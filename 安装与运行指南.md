# Agent 身份识别与授权系统 — 安装与运行指南

## 1. 环境要求

| 组件 | 最低版本 | 推荐版本 |
|------|----------|----------|
| Python | 3.10+ | 3.11+ |
| pip | 23.0+ | 最新 |
| Git | 2.30+ | 最新 |
| 操作系统 | Windows 10 / macOS 12 / Ubuntu 20.04 | Windows 11 / macOS 14 |

## 2. 安装步骤

### 2.1 克隆项目

```bash
cd <工作目录>
git clone <仓库地址> Agent身份识别系统2.2
cd Agent身份识别系统2.2
```

### 2.2 创建虚拟环境

```bash
# Windows
python -m venv venv
venv\Scripts\activate

# macOS / Linux
python3 -m venv venv
source venv/bin/activate
```

### 2.3 安装依赖

```bash
pip install -r requirements.txt
```

**核心依赖清单：**

| 包名 | 版本 | 用途 |
|------|------|------|
| fastapi | 0.135.3 | Web 框架 |
| uvicorn[standard] | 0.24.0 | ASGI 服务器 |
| PyJWT | 2.8.0 | JWT 认证 |
| bcrypt | >=4.0.0 | 密码哈希 |
| PyNaCl | >=1.5.0 | Ed25519 加密 |
| websockets | 16.0 | WebSocket 支持 |
| pydantic | 2.13.0 | 数据验证 |
| pydantic-settings | 2.6.1 | 配置管理 |
| sqlalchemy | 2.0.36 | ORM |
| httpx | >=0.25.0 | HTTP 客户端 |
| lark-oapi | >=1.4.0 | 飞书 SDK |

### 2.4 配置环境变量

复制示例配置文件并编辑：

```bash
cp .env.example .env
```

编辑 `.env` 文件，填入实际配置：

```ini
# ===== 必需配置 =====
JWT_SECRET=your-secret-key-at-least-32-bytes-long

# ===== 飞书集成（可选，不配置则使用 Mock 模式）=====
FEISHU_APP_ID=cli_xxxxxxxx
FEISHU_APP_SECRET=xxxxxxxxxxxxxxxxxxxxxxxx
FEISHU_VERIFICATION_TOKEN=xxxxxxxxxxxxxxxxxxxxxxxx
FEISHU_ENCRYPT_KEY=xxxxxxxxxxxxxxxxxxxxxxxx

# ===== 飞书多维表格（可选）=====
BITABLE_FINANCE_APP_TOKEN=xxxxxx
BITABLE_FINANCE_TABLE_ID=xxxxxx
BITABLE_HR_APP_TOKEN=xxxxxx
BITABLE_HR_TABLE_ID=xxxxxx

# ===== LLM API（至少配置一个）=====
DEEPSEEK_API_KEY=sk-xxxxxxxxxxxxxxxxxxxxxxxx
QWEN_API_KEY=sk-xxxxxxxxxxxxxxxxxxxxxxxx

# ===== 服务器配置 =====
HOST=127.0.0.1
PORT=8000
```

### 2.5 初始化数据库

数据库在首次启动时自动初始化（SQLite），无需手动操作。

## 4. 飞书集成配置（可选）

### 4.1 创建飞书应用

1. 访问 [飞书开放平台](https://open.feishu.cn/app)
2. 点击「创建企业自建应用」
3. 记录 App ID 和 App Secret

### 4.2 配置事件订阅

1. 进入应用 → 事件与回调 → 事件订阅
2. 选择「使用长连接接收事件」
3. 添加事件：`im.message.receive_v1`
4. 保存配置

### 4.3 配置权限

在「权限管理」中开通以下权限：

| 权限 | 权限标识 |
|------|----------|
| 获取与发送单聊、群组消息 | im:message |
| 读取飞书文档 | docs:doc:readonly |
| 读取多维表格 | bitable:app:readonly |
| 读取日历 | calendar:calendar:readonly |

### 4.4 启动顺序

1. 先启动服务（`python main.py`）
2. 等待飞书 WebSocket 连接建立（控制台显示 `Feishu WS bridge: connected`）
3. 在飞书开放平台点击「保存配置」

## 7. 常见问题

### Q1: 端口 8000 被占用

```bash
# Windows
netstat -ano | findstr :8000
taskkill /PID <PID> /F

# macOS / Linux
lsof -i :8000
kill -9 <PID>
```

### Q2: 飞书 WebSocket 连接失败

1. 确认 FEISHU_APP_ID 和 FEISHU_APP_SECRET 已正确配置
2. 确认飞书应用已发布（至少发布到测试企业）
3. 确认事件订阅选择了「长连接」模式
4. 重启服务后等待 10 秒再在飞书后台保存配置

### Q3: LLM API 调用失败

1. 确认至少配置了一个 LLM API Key
2. 检查网络是否能访问对应的 API 端点
3. 系统会按优先级尝试多个 LLM：DeepSeek → Qwen → 豆包 → Mock

### Q4: 数据库锁定错误

SQLite 在高并发下可能出现锁定，重启服务即可恢复。生产环境建议切换到 PostgreSQL。

### Q5: 重置系统状态

```bash
curl -X POST http://127.0.0.1:8000/api/delegate/trust/reset
curl -X POST http://127.0.0.1:8000/api/governance/reset-all
```
