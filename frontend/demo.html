🔑 Access Token 数据结构 JWT

📋 JWT Payload 字段

字段必填说明
sub必填Token主体，格式 agent:{agent_id}
iss必填签发者，固定 agent-iam-system
aud必填受众，固定 agent-delegation
exp必填过期时间（Unix timestamp）
iat必填签发时间
jti必填JWT ID，防重放唯一标识
delegated_user必填被委托用户 user:{user_id}
capabilities必填能力声明列表 string[]
chain_depth必填委派链深度（0-5）
parent_jti可选父Token的JTI
delegated_by可选委派者Agent ID
scope可选有效权限（用户∩Agent能力）
platform可选来源平台 web/feishu
ip_address可选绑定IP地址
max_uses可选最大使用次数

🕐 Agent 超时场景

当某个Agent响应超时时，系统如何兜底处理：

1
doc_agent 委派 data_agent
正常签发委派Token，发起调用
2
data_agent 响应超时
超过设定时间阈值未返回结果
3
系统兜底：降级处理
使用缓存数据 / 跳过该步骤 / 通知用户
4
doc_agent 继续汇总
使用可用结果生成报告，标注超时部分

🛡️ 安全异常处理矩阵

异常场景触发条件系统响应审计记录
Token盗用IP绑定校验失败 / 使用次数超限拒绝请求 + Token失效 + 信任分-0.15decision=deny, reason=ip_mismatch/usage_exceeded
Token重放一次性Token重复使用拒绝请求 + 标记Token已用decision=deny, reason=token_already_used
越权委派委派能力超出父Agent权限拒绝委派 + 信任分-0.15decision=deny, reason=cascade_deny
链深度超限委派链深度 > 5拒绝委派decision=deny, reason=chain_depth_exceeded
信任分崩溃trust_score ≤ 0.3自动封禁Agent + 所有Token失效decision=auto_revoke, auto_revoked=true
频繁越权10分钟内3次拒绝自动挂起Agentdecision=auto_suspend
Agent超时30秒无响应降级处理（缓存/跳过）decision=degraded, fallback=cached
Agent失败执行异常错误隔离 + 跳过decision=error, fallback=skipped
Prompt注入风险评分 > 0.7拦截请求 + 信任分-0.15decision=deny, attack_types=[injection]
哈希链断裂审计日志被篡改完整性校验失败告警verify_chain_integrity() → false

🔄 边界流程处理

边界条件处理策略设计原则
父Agent无操作能力交集拒绝委派（除非有通配符*）最小权限原则
委派Token过期拒绝 + 要求重新签发短期Token + 自动过期
Agent被删除后Token仍有效验证Agent状态，已删除则拒绝身份-Token联动校验
并发相同请求消息去重（30秒窗口）幂等性保证
冷却期后恢复信任分+0.1恢复，低于0.3不可恢复有限宽容 + 不可逆封禁
跨模型Agent接入统一IAM接口 + 异构LLM适配器协议标准化
WebSocket断连SDK自动重连 + 状态检测高可用连接
数据库不可用内存降级 + 重启后恢复双写保障
🧠 领域认知 Insight

⚡ 传统IAM vs Agent IAM

维度传统IAM（人类用户）Agent IAM（本系统）
身份粒度用户级 / 角色级Agent级 + 委派链级 + 能力级
权限模型静态RBACRBAC + 动态委派 + 信任评分
权限变更管理员手动调整自动降级/封禁/冷却恢复
行为可预测性高（人类遵循流程）低（LLM输出不可控）→ 需实时监控
攻击面密码泄露 / 权限提升+ Prompt注入 / Token盗用 / 委派链攻击
审计需求操作日志不可篡改哈希链 + 实时完整性校验
Token生命周期长期有效（小时~天）短期 + 一次性 + 使用次数限制
跨系统协作SSO / OAuthA2A协议 + 委派Token + 能力交集
